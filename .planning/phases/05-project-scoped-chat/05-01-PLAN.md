<objective>
Wire `conversation.project_id` through `runChatTurn` so chat inside a project workspace gets:

1. `project.context_summary` injected into the system prompt
2. All linked project documents seeded as primary evidence (`PROJECT-DOC-N`)
3. Hybrid search results filtered to exclude PRIVATE docs from OTHER projects (additive rule per CONTEXT.md)
4. Memory retrieval that boosts project-tagged memories
5. New memories tagged with the conversation's project_id

The global chat flow (when `project_id` is null) is unchanged.

Output: Edits to two files. End-to-end smoke test confirms a project-tagged chat sees its project docs in the evidence pack and excludes another project's private docs.
</objective>

**Constraints:**
- No schema changes — `conversation_memory.project_id` and `conversations.project_id` already exist
- No SQL function changes to `hybrid_search` — exclusion is a post-filter in chat-turn
- No new tools — the 5 existing Claude tools work as-is; they get project context from the system prompt
- The global path (project_id IS NULL) must be behaviorally unchanged
- Project docs are seeded as `PROJECT-DOC-N` evidence labels — distinct from `DOC-N` general retrieval and `PINNED-N` user-pinned
- Exclusion: PRIVATE docs linked to ANY other project are filtered out of search results
- Memory: boost project-tagged memories in the score, don't hard-restrict (we still want global "VC won't go below 18%" type memories)
</context>

<tasks>

<task type="auto">
  <name>Task 1: retrieveRelevantMemories accepts projectId</name>
  <files>src/lib/memory.ts</files>
  <action>
Extend `retrieveRelevantMemories(userMessage, excludeConversationId, maxResults)` with an optional `projectId` parameter. When provided, project-tagged memories get a +3 score boost during ranking. Memories tagged with a DIFFERENT project_id get a -2 penalty (don't show another project's memories). Global memories (project_id IS NULL) stay neutral.

Also extend `storeMemories(memories, conversationId)` to optionally accept a `projectId` and write it onto each row.
  </action>
</task>

<task type="auto">
  <name>Task 2: runChatTurn loads project, injects context, scopes evidence</name>
  <files>src/lib/chat-turn.ts</files>
  <action>
At the top of `runChatTurn`:

1. Fetch the conversation row to get `project_id`
2. If set, fetch the project row (for `name`, `context_summary`, `description`, `color`)
3. Fetch `project_documents` for THIS project → `projectDocIds`
4. Compute `excludedDocIds`: PRIVATE docs linked to ANY OTHER project

Then in retrieval:

5. After `hybridSearch`, filter results to exclude any chunk whose `documentId` is in `excludedDocIds`
6. Pre-fetch project-doc chunks (top hybrid-search hits restricted to `projectDocIds` using the existing `documentIds` option, matchCount: 6, no rerank needed) → label as `PROJECT-DOC-N` and inject as the FIRST block of the evidence pack

In the system prompt:

7. When in a project, prepend a "PROJECT CONTEXT" block to both deep and casual prompts: project name, optional context_summary, list of counterparties, "you are working inside this project — its documents are the first DOC blocks below"

In memory:

8. Pass `projectId` to `retrieveRelevantMemories` so project memories rank first
9. Pass `projectId` to `storeMemories` (in the fire-and-forget extraction at the end)

The global path is preserved by gating every new code path on `if (projectId)`.
  </action>
</task>
