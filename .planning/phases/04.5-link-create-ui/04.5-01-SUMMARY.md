# Phase 04.5 Summary

**Two missing UI affordances unblock daily workspace use without curl**

The workspace tabs from Phase 04 had no buttons to actually link a doc or create a negotiation — both required curl. This plan adds them.

## Accomplishments

- `LinkDocumentDialog` — searches via `/api/picker?q=` with debounce, multi-select with checkmarks, role picker (primary/reference/supporting), POSTs to `/api/projects/[slug]/documents`
- `CreateNegotiationDialog` — name field (required), counterparty picker (optional, fetches from `/api/picker` filtered to companies), status pills, POSTs to `/api/negotiations`
- Documents tab gains a `+ Link document` button (header when populated, CTA in empty state) with refreshKey-driven re-fetch
- Negotiations tab gains a `+ New negotiation` button with the same pattern

## Verification

- `npx tsc --noEmit` clean
- Workspace page renders, picker returns docs + companies, link API returns `{added: 1}`, create API returns the new negotiation
- Test data cleaned up

## Task Commit

- `346f1d4` — feat(04.5): Link & Create UI affordances

## Decisions Made

1. **No server actions** — both dialogs hit the existing Phase 03 REST endpoints directly. The dialogs are small enough that the wrapping isn't worth the boilerplate.
2. **`refreshKey` state pattern** — incrementing a counter forces the tab's `useEffect` to re-fetch. Avoids prop-drilling state through the dialog.
3. **Multi-select for documents, single for negotiations** — link is a bulk operation; create is single-purpose.

## Issues / Deviations

None — straightforward composition of existing primitives.

## Next Phase Readiness

Workspace is fully usable from the UI. Ready for Phase 05 (project-scoped chat backend).
